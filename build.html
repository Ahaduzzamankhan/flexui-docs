Build System

FlexUI uses esbuild as its bundler — delivering sub-second builds with code splitting, tree-shaking, and automatic asset hashing.

Build Pipeline

Running flexui build executes the following steps:

  1. Load flexui.config.js via defineConfig
  2. Resolve all page entry points from pagesDir
  3. Bundle with esbuild — code splitting at dynamic import() boundaries
  4. Hash output filenames for cache-busting
  5. Write asset-manifest.json mapping source paths to output paths
  6. If ssg: true, pre-render each page route to static HTML
  7. Copy publicDir files to outDir

Code Splitting

FlexUI automatically splits bundles at every dynamic import() call. This includes:

  • Components wrapped in lazy()
  • Manual import('...') calls in your code
  • Route components loaded by the file router
// Each of these creates a separate chunk
const Dashboard = lazy(() => import('./pages/Dashboard.js'));
const chart = await import('./lib/chart.js');

Tree Shaking

esbuild's tree-shaker removes any exports that are never imported. This applies to FlexUI's own modules — if you don't use generateSitemap, it won't appear in your bundle.

Side-effect free
FlexUI's source modules have no top-level side effects, which means esbuild can aggressively eliminate unused exports. Keep your own modules side-effect free for best results.

Asset Manifest

After a build, dist/asset-manifest.json maps source entry points to their hashed output paths:

{
  "src/main.js":              "static/js/main.a3f9c.js",
  "pages/index.js":          "static/js/pages/index.b7d21.js",
  "pages/blog/[slug].js":    "static/js/pages/blog/slug.cc410.js"
}

Use the manifest in your SSR server to inject the correct script tags:

import manifest from '../dist/asset-manifest.json' assert { type: 'json' };
const mainBundle = manifest['src/main.js'];

HMR Dev Server

The dev server rebuilds only changed modules using esbuild's incremental API and pushes updates to the browser via WebSocket.

// Injected into every page by the dev server
const ws = new WebSocket('ws://localhost:3000/__hmr');
ws.onmessage = ({ data }) => {
  const { type, path } = JSON.parse(data);
  if (type === 'update') hotReplace(path);
  if (type === 'reload')  location.reload();
};

Build errors are surfaced as a full-page error overlay with the source file and line number highlighted.
