Configuration

All FlexUI settings live in flexui.config.js at the root of your project. Use defineConfig for full TypeScript intellisense.

defineConfig

Wrap your config in defineConfig for type checking and IDE autocomplete:

flexui.config.js 
import { defineConfig } from 'flexui/build';

export default defineConfig({
  entry:     'src/main.js',
  pagesDir:  'pages',
  publicDir: 'public',
  outDir:    'dist',
  port:      3000,
  host:      'localhost',
  ssg:       false,
  ssr:       false,
  baseUrl:   'https://myapp.com',
  minify:    true,
  sourcemap: true,
  target:    ['es2020', 'chrome90', 'firefox90', 'safari14'],
  env: {
    APP_NAME: 'MyApp',
  },
});

Rendering Modes

Set ssr or ssg to true to switch rendering strategy:

// Static Site Generation
defineConfig({ ssg: true, baseUrl: 'https://mysite.com' });

// Server-Side Rendering
defineConfig({ ssr: true, port: 8080 });
Only one at a time
Do not set both ssr and ssg to true in the same config. Choose one rendering strategy per project.

Environment Variables

Variables declared in env are injected at build time and available via import.meta.env.*:

// flexui.config.js
env: { API_URL: 'https://api.example.com' }

// In your components
const url = import.meta.env.API_URL;
Runtime vs Build-time
These variables are replaced at build time (like process.env in webpack). They are not accessible at runtime on the server unless you also pass them through server config.

Build Options

The target array maps directly to esbuild's target option. Control browser compatibility:

// Modern browsers only
target: ['es2022', 'chrome100']

// Wider compatibility
target: ['es2018', 'chrome80', 'firefox78', 'safari13']
