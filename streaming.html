Streaming SSR

Traditional SSR waits for all data before sending HTML. Streaming SSR sends the page shell immediately, then streams deferred chunks as async operations complete. This dramatically reduces Time To First Byte (TTFB).

Requires a streaming-capable server
Use renderToStream or renderStreaming with Node.js HTTP, Deno, or any runtime that supports ReadableStream.

renderStreaming

Returns a ReadableStream that flushes the shell immediately and injects deferred content via inline <script> chunks as Suspense boundaries resolve.

import { renderStreaming } from 'flexui/server';

export async function handler(req, res) {
  const stream = renderStreaming(
    createElement(App, { url: req.url })
  );

  res.setHeader('Content-Type', 'text/html');
  res.setHeader('Transfer-Encoding', 'chunked');

  const reader = stream.getReader();
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    res.write(value);
  }
  res.end();
}

Streaming with Suspense

Wrap slow data-fetching components in Suspense. The shell renders immediately; deferred content streams in when ready.

Island Architecture

For mostly-static pages, you can skip hydrating the entire page. Instead, mark only interactive components as Islands — only they receive JavaScript on the client. Everything else stays as plain HTML.

This delivers the fastest possible page load: static HTML with targeted interactivity.

Static HTML shell   ──── sent immediately, no JS needed
       │
       ├── 🏝 Island: <SearchBar />   ← hydrated
       ├── 🏝 Island: <VideoPlayer /> ← hydrated
       └── Everything else             ← static, zero JS cost

Island Component

Wrap any interactive component in <Island>. The server renders it to HTML; the client hydrates only that subtree.

pages/index.js (SSG/SSR) 
import { createElement } from 'flexui';
import { Island } from 'flexui/server';
import SearchBar from '../components/SearchBar.js';

export default function HomePage() {
  return createElement('main', null,
    createElement('h1', null, 'Welcome'),          // static
    createElement('p', null, 'Lots of static content...'), // static
    createElement(Island, { component: SearchBar, props: { placeholder: 'Search' } })
    // ↑ only this gets hydrated client-side
  );
}

hydrateIslands

Call this once in your client entry point. It scans the DOM for island markers and hydrates each one independently.

src/main.js (client) 
import { hydrateIslands } from 'flexui/server';

// Pass a map of component name → component module
hydrateIslands({
  SearchBar: () => import('./components/SearchBar.js'),
  VideoPlayer: () => import('./components/VideoPlayer.js'),
});
Islands ship zero JS by default
Components not wrapped in <Island> ship no JavaScript to the browser. Use islands only for genuinely interactive components — forms, media players, live data feeds.
