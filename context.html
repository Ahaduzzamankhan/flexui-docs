Context

Context provides a way to pass data through the component tree without manually threading props at every level. Use it for themes, auth, locale, and other global state.

createContext

Creates a context object with a default value used when no Provider is found above in the tree.

const MyCtx = createContext(defaultValue)
PropertyDescription
MyCtx.ProviderComponent that supplies a value to the subtree
MyCtx._currentValueCurrent context value (internal)

Provider

Wrap any part of the tree in a Provider to supply a value. All consumers in the subtree re-render when value changes.

createElement(MyCtx.Provider, { value: currentValue },
  children
)
Reference identity matters
If you pass an object literal as value, it creates a new reference on every render, causing all consumers to re-render. Memoize the value object with useMemo.

useContext

Reads the nearest Provider value above in the tree. Automatically subscribes to updates.

const value = useContext(MyCtx);

Full Pattern

The recommended pattern is to encapsulate context creation, the provider, and custom hooks in a single module:

context/AuthContext.js 
import { createContext, createElement } from 'flexui';
import { useContext, useState, useMemo } from 'flexui/hooks';

const AuthCtx = createContext(null);

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);

  const value = useMemo(() => ({
    user,
    login:  (u) => setUser(u),
    logout: () => setUser(null),
  }), [user]);

  return createElement(AuthCtx.Provider, { value }, children);
}

// Custom hook — consumers use this, not AuthCtx directly
export function useAuth() {
  const ctx = useContext(AuthCtx);
  if (!ctx) throw new Error('useAuth must be used inside AuthProvider');
  return ctx;
}
pages/profile.js 
import { useAuth } from '../context/AuthContext.js';

function ProfilePage() {
  const { user, logout } = useAuth();
  return createElement('div', null,
    createElement('p', null, `Welcome, ${user.name}`),
    createElement('button', { onClick: logout }, 'Sign out')
  );
}

Performance Notes

  • Every consumer of a context re-renders when its Provider value changes.
  • Split large contexts into smaller, focused ones to avoid unnecessary re-renders.
  • Wrap the value object in useMemo to prevent a new reference being created every render cycle.
  • For very high-frequency updates (e.g. mouse position), prefer a ref or a dedicated state manager instead of context.
