Lifecycle

Understand when FlexUI mounts, updates, and unmounts components — and how to hook into each phase from both functional and class components.

Lifecycle Phases

Mount          → Update          → Unmount
─────────────────────────────────────────
render()         render()          componentWillUnmount()
    ↓                ↓
DOM written      DOM patched
    ↓                ↓
componentDidMount()  componentDidUpdate()

Functional Components

Use useEffect to hook into lifecycle events from functional components:

Mount

useEffect(() => {
  // Runs once after the component is added to the DOM
  const subscription = subscribe(handler);

  // Return a cleanup function — runs on unmount
  return () => subscription.unsubscribe();
}, []); // empty deps = mount + unmount only

Update

useEffect(() => {
  // Runs after every render where `userId` changed
  fetchProfile(userId).then(setProfile);
}, [userId]);

Every render

useEffect(() => {
  document.title = `Page: ${title}`;
}); // no deps array = runs after every render

Before paint (useLayoutEffect)

useLayoutEffect(() => {
  // Runs synchronously after DOM mutations, before browser paint
  // Use to measure DOM dimensions or imperatively sync layout
  const { height } = boxRef.current.getBoundingClientRect();
  setBoxHeight(height);
}, []);

Class Components

MethodPhaseNotes
constructor(props)MountInitialize state here. Call super(props) first.

componentDidUpdate(pP, pS)UpdatePrevious props/state are passed. Guard updates with comparisons to avoid infinite loops.

Full class lifecycle example 
class DataLoader extends Component {
  state = { data: null, loading: true };
  _controller = null;

  async componentDidMount() {
    this._controller = new AbortController();
    const data = await fetchData({ signal: this._controller.signal });
    this.setState({ data, loading: false });
  }

  componentDidUpdate(prevProps) {
    if (prevProps.id !== this.props.id) {
      this.setState({ loading: true });
      this.componentDidMount(); // re-fetch on id change
    }
  }

  componentWillUnmount() {
    this._controller?.abort(); // cancel in-flight request
  }

  render() {
    if (this.state.loading) return createElement('p', null, 'Loading…');
    return createElement('div', null, JSON.stringify(this.state.data));
  }
}

Scheduler & Priority

FlexUI's cooperative scheduler processes work in priority order, yielding to the browser every 5ms to preserve frame rate.

PriorityValueUse for
Priority.IMMEDIATE1Synchronous work — user input handlers, error recovery
Priority.HIGH2Animations, transitions, re-renders
Priority.NORMAL3State updates (default)
Priority.LOW4Prefetch, background data loading
Priority.IDLE5Analytics, non-critical side effects
 
import { scheduleWork, flushSync, batchUpdates, Priority } from 'flexui';

// Schedule non-critical work
scheduleWork(() => sendAnalytics(event), Priority.IDLE);

// Force synchronous flush (use sparingly)
flushSync(() => setState(newValue));

// Batch multiple setState calls into one render
batchUpdates(() => {
  setCount(c => c + 1);
  setName('updated');
});
