Virtual DOM

FlexUI maintains a lightweight virtual DOM tree in memory. On each render it diffs the new tree against the previous one and applies only the minimal set of DOM mutations needed.

How It Works

Every call to createElement returns a plain JavaScript object called a FlexElement:

{
  $$typeof: Symbol(FlexElement),
  type:     'button',
  key:      null,
  ref:      null,
  props: {
    className: 'btn',
    onClick:   [Function],
    children:  'Click me',
  },
}

These objects are cheap to create and never touch the real DOM directly. The renderer compares the new virtual tree to the previous one and generates a minimal list of patch operations.

Diffing Algorithm

FlexUI uses a two-phase diff:

  1. Type check — if the element type changed (e.g. 'div' → 'span'), the old node is unmounted and a new one is mounted.
  2. Props diff — for elements of the same type, only changed attributes and event handlers are updated.
  3. Children diff — children are reconciled by index, or by key if provided (see below).

The patch applicator then makes the minimal number of DOM calls: setAttribute, removeAttribute, addEventListener, removeChild, appendChild, etc.

Keys

When rendering lists, provide a stable key prop so FlexUI can match old and new nodes correctly. Without keys, all items after a deletion or insertion are unnecessarily updated.

Never use array index as key for dynamic lists
If items can be reordered, added, or removed, using the array index as a key causes incorrect reconciliation — state is carried over to the wrong elements. 
// ✗ Bad — index key causes bugs on reorder/delete
items.map((item, i) => createElement(Row, { key: i }, item.name));

// ✓ Good — stable ID from data
items.map(item => createElement(Row, { key: item.id }, item.name));

Event System

FlexUI uses event delegation rather than attaching individual listeners to every DOM node. A single listener is registered on the root container for each event type. When an event fires, it bubbles up to the root and FlexUI dispatches it through the virtual tree.

All event handlers receive a SyntheticEvent — a normalized wrapper that irons out cross-browser differences and ensures consistent behaviour.

createElement('input', {
  onChange: (e) => {
    // e is a SyntheticEvent
    console.log(e.target.value);
    e.stopPropagation();  // works as expected
    e.preventDefault();   // works as expected
  }
});

Supported Event Props

All standard DOM events are supported using camelCase prop names: onClick, onChange, onInput, onSubmit, onKeyDown, onKeyUp, onFocus, onBlur, onMouseEnter, onMouseLeave, onScroll, and many more.

isValidElement

Returns true if the value is a FlexElement (created by createElement).

import { isValidElement } from 'flexui';

isValidElement(createElement('div', null))  // → true
isValidElement('hello')                    // → false
isValidElement(null)                       // → false
