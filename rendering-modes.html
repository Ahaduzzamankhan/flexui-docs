CSR / SSR / SSG

FlexUI supports three rendering modes — Client-Side Rendering, Server-Side Rendering, and Static Site Generation. Each mode is configured in flexui.config.js and can be mixed per-page.

Client-Side Rendering (CSR)

The default mode. The server sends a minimal HTML shell; FlexUI boots in the browser and renders the full UI.

src/main.js 
import { createRoot } from 'flexui';
import App from './App.js';

createRoot(document.getElementById('root')).render(createElement(App, null));

You can also hydrate a server-rendered HTML string on the client:

import { hydrateRoot } from 'flexui';
hydrateRoot(document.getElementById('root'), createElement(App, null));

Server-Side Rendering (SSR)

Enable SSR in config, then use renderToString or renderToStream in your server handler.

flexui.config.js 
export default defineConfig({ ssr: true, port: 8080 });
server/handler.js 
import { renderToString } from 'flexui/server';
import App from '../src/App.js';

export async function handler(req, res) {
  const html = await renderToString(
    createElement(App, { url: req.url })
  );

  res.send(`<!DOCTYPE html>
<html><body><div id="root">${html}</div></body></html>`);
}

renderToString vs renderToStream

FunctionReturnsUse when
renderToString(element)Promise<string>Simpler setup, full HTML before send
renderToStream(element)ReadableStreamFaster TTFB, streaming Suspense boundaries
renderToDocument(element)Promise<string>Full HTML document with injected <head> tags

Static Site Generation (SSG)

Pages are pre-rendered at build time and output as static HTML files. No server required at runtime.

flexui.config.js 
export default defineConfig({ ssg: true, baseUrl: 'https://mysite.com' });
## Build command
flexui build --ssg

getServerSideProps

Export this function from any page to fetch data on every request in SSR mode. The returned props are passed directly to your page component.

pages/dashboard.js 
export async function getServerSideProps({ req, params, query }) {
  const user = await getUser(req.cookies.token);
  if (!user) return { redirect: '/login' };

  return { props: { user } };
}

export default function Dashboard({ user }) {
  return createElement('h1', null, `Hi, ${user.name}`);
}

getStaticProps & getStaticPaths

Used in SSG mode. getStaticProps fetches data at build time. For dynamic routes, getStaticPaths declares which paths to pre-render.

pages/blog/[slug].js 
export async function getStaticPaths() {
  const posts = await fetchAllPosts();
  return {
    paths: posts.map(p => ({ params: { slug: p.slug } })),
    fallback: false, // 404 for unknown slugs
  };
}

export async function getStaticProps({ params }) {
  const post = await fetchPost(params.slug);
  return { props: { post } };
}

export default function Post({ post }) {
  return createElement('article', null,
    createElement('h1', null, post.title),
    createElement('div', { dangerouslySetInnerHTML: { __html: post.content } })
  );
}
Fallback: false vs true
fallback: false returns a 404 for any path not in the paths list. fallback: true serves a loading state while the page is generated on-demand and cached for future requests.
